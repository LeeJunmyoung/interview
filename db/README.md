## Lock 이란
> 트랜잭션 처리의 순차성을 보장하기 위한 방법.  
> 트랜잭션 처리능력이 핵심적인 요소.  
> 같은 자원을 액세스하려는 다중 트랜잭션 환경에서 데이터베이스의 일관성, 무결성 유지를 위해 트랜잭션의 순차적 진행을 보장하는 직렬화(serialization) 장치가 필요.  
> DBMS 마다 Lock 을 구현하는 세부 기능이 다름.
